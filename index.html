How It Works
From login to enforcement in seconds
The plugin hooks into OpenClaw's gateway lifecycle â€” authentication, policy fetch, and enforcement happen automatically on startup.

Authenticate

User runs /clawguard-login. Browser opens to your IdP. PKCE code exchange returns a ClawGuard session token.

Fetch Policy

Plugin fetches the org policy from the control plane. Cached locally with TTL and stale fallback if the server is unreachable.

Enforce Tools

Every tool call hits the enforcer hook. Allowed tools proceed; denied tools are blocked and logged to the audit trail.

Filter Skills

Only admin-approved skills are loaded. Org-wide or per-user scope. Unapproved skills are invisible to the gateway.

Log Everything

Tool calls, sessions, and LLM interactions are batched and flushed to the control plane. Queryable from the admin console.

Stay in Control

Heartbeat polling checks for kill switch state and policy updates. If the control plane is down, gateways fail secure.

Quick Start
Up and running in minutes
Three components: a PostgreSQL-backed API server, a Next.js admin console, and an OpenClaw plugin. All configured through environment variables.
Control Plane
# Start the API server cd packages/clawguard-server export DATABASE_URL="postgresql://localhost:5432/clawguard" export JWT_SECRET="your-secret-here" export CORS_ORIGIN="http://localhost:4200" # Run migrations pnpm db:migrate # Start server (port 4100) pnpm dev
OpenClaw Plugin
